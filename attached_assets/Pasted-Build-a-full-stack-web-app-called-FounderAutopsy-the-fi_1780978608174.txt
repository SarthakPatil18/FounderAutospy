Build a full-stack web app called FounderAutopsy — the first platform where founders document their own startup shutdowns, and AI performs a structured autopsy so the next founder doesn't repeat the same mistake.

Tagline: "What killed the company. Precisely."
Domain intent: founderautopsy.com

── WHAT MAKES THIS DIFFERENT (USPs — build every one of these) ──────────────

USP 1 — Founder-submitted postmortems (PRIMARY)
  No editorial team, no scraping TechCrunch.
  The actual founder writes their own shutdown story.
  This is primary source data. Nobody else has it.
  Every postmortem page must feel like a founder wrote it, not a journalist.

USP 2 — AI autopsy engine (PRIMARY)
  After submission, send the full postmortem to OpenAI GPT-4o.
  AI produces a structured "autopsy report":
    - Primary cause of death (1 sentence)
    - Contributing factors (3-5 bullet points)
    - Failure category tags from: [PMF mismatch, Cash flow, Team conflict,
      Market timing, Competition, Tech debt, Regulatory, Execution,
      Customer acquisition, Pivot failure]
    - Autopsy verdict (2-3 sentence summary, clinical tone)
  Display this in a dark code-editor-style card on the postmortem page.
  Label it "/ autopsy report" in monospace above the card.

USP 3 — Similar failure matching
  After AI tagging, query the DB for other startups with overlapping failure tags.
  Show "3 startups that died the same way" at the bottom of every postmortem.
  Each matched card shows: name, industry, year, shared failure tag highlighted.

USP 4 — Free and open forever
  No paywall. No login required to read.
  Login only required to submit a postmortem.
  Add a visible line on the homepage: "Free forever. No paywall. Community-built."

── AI AUTOPSY ENGINE ────────────────────────────────────────────────────────

Trigger: after a postmortem is submitted and saved to DB

API call to OpenAI GPT-4o:

System prompt:
  "You are an expert startup analyst performing a clinical autopsy on a failed startup.
   Be precise, analytical, and brutally honest. Do not sugarcoat.
   Your tone is that of a forensic investigator, not a motivational speaker.
   Respond ONLY in valid JSON. No markdown, no preamble."

User prompt:
  "Startup: [name]
   Industry: [industry]
   Founded: [year] | Closed: [year]
   Story: [story text]
   What failed: [what_failed text]
   Lessons: [lessons_learned text]
   Metrics: Peak MRR $[x], Team [x], Raised $[x]

   Return this exact JSON structure:
   {
     'primary_cause': 'one sentence',
     'contributing_factors': ['factor 1', 'factor 2', 'factor 3'],
     'tags': ['tag1', 'tag2'],
     'verdict': '2-3 sentence clinical summary'
   }

   Tags must be chosen ONLY from:
   PMF mismatch, Cash flow, Team conflict, Market timing, Competition,
   Tech debt, Regulatory, Execution, Customer acquisition, Pivot failure"

On response: parse JSON, save all fields to the startups table, then run
similar failure matching by querying for startups sharing >=1 tag.
